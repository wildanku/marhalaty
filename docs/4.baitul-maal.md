# Task 5: Crowdfunding & Baitul Maal (Polymorphic Ledger)

## 1. Context & Business Logic
The **Baitul Maal** module handles two streams of fundraising: Always-on global funds (e.g., Zakat, General Sedekah) and Incident-based Campaigns (e.g., "Help Sick Alumni", "Build Mosque"). 

### Key Features:
1. **Polymorphic Ledger:** A single `donations` table to record all incoming funds, regardless of whether they are for a global fund or a specific campaign.
2. **Data Integrity:** Strict use of Database Transactions to ensure balance updates and donation records are perfectly synchronized.
3. **Transparency & Privacy:** * "Hamba Allah" (Anonymous) toggle for donors.
   * "Campaign Updates" feature to post proof of fund distribution.
4. **Gamification & Utility:** Visual progress bars for campaigns and a mini Zakat Calculator.

---

## 2. Implementation Task for AI Agent
**Instruction for AI Agent:** Generate the migrations, models, controllers, and React components within the `app/Domains/Donation` directory utilizing polymorphic relationships.

### A. Backend: Database & Models
1. **Funds Table (`funds`):** (For always-on donations like Zakat)
   * `name` (string), `slug` (string), `description` (text)
   * `collected_amount` (decimal/integer, default: 0)
2. **Campaigns Table (`campaigns`):** (For specific events)
   * `title`, `slug`, `description`, `image_url`
   * `target_amount` (decimal/integer)
   * `collected_amount` (decimal/integer, default: 0)
   * `end_date` (date, nullable)
   * `status` (enum: `active`, `completed`, `draft`)
3. **Campaign Updates Table (`campaign_updates`):**
   * `campaign_id` (foreign key)
   * `title`, `content` (text)
4. **Donations Table (`donations`):** (The Polymorphic Ledger)
   * `donatable_id` (unsignedBigInteger)
   * `donatable_type` (string) -> e.g., `App\Domains\Donation\Models\Campaign`
   * `user_id` (foreign key)
   * `amount` (decimal/integer)
   * `status` (enum: `pending`, `paid`, `failed`, default: `pending`)
   * `is_anonymous` (boolean, default: false)

### B. Backend: Controllers (`App\Domains\Donation\Controllers`)
1. **`CampaignController@show`:**
   * Load the campaign along with its `campaign_updates` and the latest 10 `paid` donations.
   * **Data Masking:** If a donation has `is_anonymous == true`, replace the `user->name` with "Hamba Allah" before sending it to Inertia.
2. **`DonationController@store`:**
   * Receive `amount`, `is_anonymous`, `donatable_type`, and `donatable_id`.
   * Create a `pending` record in the `donations` table.
   * *(Note: The actual increment of `collected_amount` will be handled in the Payment Webhook task using `DB::transaction()`, do not increment it here).*

### C. Frontend: React + Inertia Components
1. **Baitul Maal Hub (`Pages/Donation/Index.jsx`):**
   * **Top Section:** Quick action cards for "Zakat" and "Sedekah Umum" (routing to `Funds`).
   * **Campaign List:** Cards for active campaigns showing `title`, `target_amount`, and a calculated progress bar.
2. **Campaign Detail UI (`Pages/Donation/Show.jsx`):**
   * **Visuals:** Tailwind CSS progress bar `((collected_amount / target_amount) * 100)%`.
   * **Donor Wall:** List of recent donors. Display "Hamba Allah" if flagged anonymous.
   * **Updates Tab:** A section displaying records from `campaign_updates`.
3. **Donation Form Component:**
   * Input for amount with preset chips (e.g., Rp 50.000, Rp 100.000).
   * Checkbox: "Sembunyikan nama saya (Hamba Allah)".
   * **Zakat Calculator Modal:** A client-side Zustand/React state component that calculates 2.5% of inputted monthly income, which then auto-fills the donation amount. Styling follows the "Ijo Kukus" theme.

---

## 3. Definition of Done (DoD)

- [ ] Backend properly masks names as "Hamba Allah" for anonymous donations in the API response.
- [ ] React UI accurately renders the progress bar and handles the Zakat calculation logic on the client side.
- [ ] Donation submission creates a `pending` record linked correctly to either a Fund or a Campaign.
