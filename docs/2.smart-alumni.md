# Task 3: Smart Alumni Directory & Privacy Controls

## 1. Context & Business Logic
The **Smart Alumni Directory** is the core networking engine of the portal. It allows alumni to discover each other based on location, profession, and graduation year (Marhalah). 

### Key Features:
1. **Smart Search & Filters:** Efficient querying without reloading the page, utilizing infinite scroll for a modern UX.
2. **Privacy First:** Alumni have granular control over their data visibility (`public`, `circle/marhalah`, `private`).
3. **Data Protection:** Sensitive information like WhatsApp numbers must never be displayed as raw text to prevent scraping. It should be wrapped in an action button, and only visible to `verified` users.
4. **Alumni Business Showcase:** A dedicated section in the user profile to promote their business or profession.

---

## 2. Implementation Task for AI Agent
**Instruction for AI Agent:** Generate the models, migrations, controllers, and React components within the `app/Domains/Alumni` directory to handle the directory listing, advanced filtering, and user profile display.

### A. Backend: Database & Models
1. **Packages:** Install and configure `spatie/laravel-query-builder` (for advanced filtering) and `spatie/laravel-sluggable` (for SEO-friendly profile URLs).
2. **Database Update (Users Table or Profile Table):** Add the following fields to the user schema:
   * `slug` (string, unique)
   * `profession` (string, nullable)
   * `city` (string, nullable)
   * `privacy_setting` (enum: `public`, `circle`, `private`, default: `public`)
   * `business_showcase_url` (string, nullable - for business banners/links)
3. **Model Setup:** Update the `User` model to implement the `HasSlug` trait and configure it to generate slugs from the `name` field.

### B. Backend: Directory Controller (`App\Domains\Alumni\Controllers\DirectoryController`)
1. **`index()`:**
   * Use `Spatie\QueryBuilder\QueryBuilder` to fetch users.
   * Allowed filters: `city`, `profession`, `marhalah_year`, and a global `search` (name).
   * Apply a base query condition: Only fetch users where `privacy_setting` is NOT `private`.
   * Return paginated data to Inertia (to be handled as infinite scroll on the frontend).
2. **`show($slug)`:**
   * Fetch user by `slug`.
   * **Authorization Logic:** Check the target user's `privacy_setting`. If `circle`, ensure the currently authenticated user has the same `marhalah_year`. 
   * **Data Masking:** If the authenticated user is `is_verified = false`, mask or remove the `phone_number` from the returned Inertia payload.

### C. Frontend: React + Inertia Components
1. **Directory Listing UI (`Pages/Alumni/Index.jsx`):**
   * **Layout:** A sidebar for filters (City, Profession, Marhalah) and a main grid for Alumni Cards.
   * **UX:** Implement a debounced search input. Use an Intersection Observer or a library to handle **Infinite Scroll** by appending new pagination data to the existing state.
   * **Styling:** "Ijo Kukus" (muted green) palette using Tailwind CSS and Shadcn UI components (Cards, Badges, Inputs).
2. **Profile Detail UI (`Pages/Alumni/Show.jsx`):**
   * **Header:** Avatar, Name, Marhalah, and Profession.
   * **Business Showcase:** A dedicated card highlighting their business/profession banner if it exists.
   * **Contact Action:** Instead of showing the raw WhatsApp number, render a "Hubungi via WhatsApp" (Contact via WA) button using a `wa.me` link. Disable or hide this button if the backend masked the data (e.g., if the viewer is unverified).

---

## 3. Definition of Done (DoD)
- [ ] Users can browse the directory and filter by city, profession, and search term without full page reloads.
- [ ] Spatie Query Builder is properly integrating with the Eloquent models.
- [ ] Profile URLs are SEO-friendly using slugs (e.g., `/p/ahmad-fauzi`).
- [ ] Privacy settings are strictly enforced (e.g., unverified users cannot access WhatsApp links).
- [ ] Infinite scrolling is functional on the directory listing page.
