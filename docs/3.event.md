# Task 4: RSVP & Event Management (Event-Commerce)

## 1. Context & Business Logic
The **RSVP & Event Management** module is designed as an "Event-Commerce Hybrid." It handles event registrations alongside dynamic donations (Infak) and merchandise sales (Add-ons) in a single checkout flow.

### Key Features:
1. **Dynamic Pricing Engine:** Events can be configured as:
   * `free`: Pure RSVP.
   * `fixed`: Fixed ticket price.
   * `flexible`: Pay-What-You-Want (voluntary or mandatory infak) with optional predefined tiers (e.g., 10k, 20k, 50k) and a minimum custom amount.
2. **Add-on Catalog (Merchandise):** Committees can sell physical add-ons (like reunion t-shirts with sizes) attached to an event.
3. **Single Checkout:** The total bill equals `(RSVP/Infak Amount) + (Add-on Price * Quantity)`.
4. **Access Control:** Events can be scoped to specific audiences (e.g., only visible to a certain Marhalah or City).

---

## 2. Implementation Task for AI Agent
**Instruction for AI Agent:** Generate the migrations, models, controllers, and React components within the `app/Domains/Event` directory to handle dynamic event creation, the RSVP flow, and the merchandise cart system.

### A. Backend: Database & Models
1. **Events Table (`events`):**
   * `title`, `slug`, `description`, `location`, `event_date`
   * `payment_type` (enum: `free`, `fixed`, `flexible`)
   * `pricing_rules` (json) -> stores rules like `{"amount": 50000}` or `{"options": [10000, 20000], "allow_custom": true, "min_custom": 10000}`.
   * `visibility_scope` (string, nullable) -> to restrict to specific marhalah/region.
2. **Event Add-ons Table (`event_addons`):**
   * `event_id` (foreign key)
   * `name`, `price`, `stock_quantity`
   * `variants` (json, nullable) -> stores size options (e.g., S, M, L, XL).
3. **RSVPs/Tickets Table (`rsvps`):**
   * `event_id`, `user_id`
   * `base_amount` (decimal/integer) -> the infak/ticket amount.
   * `total_amount` (decimal/integer) -> base + addons.
   * `status` (enum: `pending`, `paid`, `expired`, `failed`)
   * `add_ons_snapshot` (json) -> records the purchased items, sizes, and quantities.
   * `qr_code_path` (string, nullable).

### B. Backend: Controllers (`App\Domains\Event\Controllers`)
1. **`EventController@index` & `show`:** * Fetch events. Apply logic to only show events relevant to the authenticated user's `marhalah_year` or `city` based on `visibility_scope`.
2. **`RsvpController@store`:**
   * **Validation:** Strictly validate the incoming dynamic pricing request based on the event's `pricing_rules` (e.g., ensure custom amount is >= `min_custom`).
   * **Cart Calculation:** Calculate `base_amount` + sum of `(addon_qty * addon_price)`.
   * **Stock Reservation:** Decrement the `stock_quantity` in `event_addons`.
   * **Creation:** Create the `Rsvp` record with `status = pending`.
   * *(Note: Actual payment gateway redirection will be handled in a separate Payment Service task).*

### C. Frontend: React + Inertia Components
1. **Event Detail UI (`Pages/Event/Show.jsx`):**
   * Hero banner, description, and Google Maps/Leaflet embed for location.
2. **Dynamic RSVP Form Component (`Pages/Event/Partials/RsvpForm.jsx`):**
   * **Data-Driven UI:** Read the `event.payment_type` and `event.pricing_rules`.
   * If `flexible`: Render selectable cards/radio buttons for predefined amounts, plus an input field for "Custom Amount" if allowed.
   * Add Zod for client-side validation (e.g., minimum infak amount).
3. **Merchandise Add-on Component:**
   * Render available add-ons with a quantity selector and variant dropdown (e.g., Size).
4. **Checkout Summary Bar:**
   * A sticky bottom bar showing the calculated Total Amount and a "Proceed to Payment / RSVP" button. Styling must follow the "Ijo Kukus" theme (muted green CTA).
