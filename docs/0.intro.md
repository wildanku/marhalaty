````markdown
# Task 1: Project Preface & Initial Scaffolding

## 1. Project Overview

**SaaS Alumni Portal Architect** is a multi-tenant ready, community-driven platform. The first implementation (Pilot Project) is dedicated to the **Dynamic Foundation Class of 2013 (Dynamic Generation)**.

### Core Pillars:

1. **Smart Alumni Directory:** A networking-first database with privacy controls.
2. **Event Management (Event-Commerce):** RSVP system with dynamic pricing and merchandise add-ons.
3. **Baitul Maal (Crowdfunding):** A transparent ledger system for Zakat, Sedekah, and specific social campaigns.

---

## 2. Technical Stack

- **Backend:** Laravel 13.x (PHP 8.5+)
- **Frontend:** React.js via Inertia.js (Single Page Application feel)
- **Styling:** Tailwind CSS + Shadcn UI (Tone: "Ijo Kukus" / Muted Green)
- **Package Manager (Frontend):** **pnpm** (Strictly use pnpm instead of npm)
- **Database:** PostgreSQL
- **Auth:** Google OAuth Only (No passwords/OTP)

---

## 3. Implementation Task: Initial Setup

**Instruction for AI Agent:** Generate the following scaffolding for a Laravel 13 project with Inertia/React and prepare the domain-driven structure.

### A. Core Scaffolding

1. Initialize a new Laravel project with **Inertia.js (React)** and **Tailwind CSS**.
2. Install **Laravel Socialite** for Google Authentication.
3. Create a domain-based directory structure under `app/`:
   - `app/Domains/Shared`
   - `app/Domains/Alumni`
   - `app/Domains/Event`
   - `app/Domains/Donation`

### B. Environment & Scoping Configuration

Add the following variables to `.env` to handle the "Single Marhalah" pilot phase:

```env
COMMUNITY_SCOPE=single
TARGET_MARHALAH_YEAR=2013
TARGET_MARHALAH_NAME="Dynamic Generation"
```
````

Create `config/community.php`:

```php
<?php

return [
    'scope' => env('COMMUNITY_SCOPE', 'global'),
    'target_marhalah' => [
        'year' => (int) env('TARGET_MARHALAH_YEAR', 2013),
        'name' => env('TARGET_MARHALAH_NAME', 'Dynamic Generation'),
    ],
];
```

### C. Database & Model Foundation (User)

Modify the default `users` table migration:

- Remove `password` and `email_verified_at`.
- Add `google_id` (string, unique).
- Add `avatar_url` (string, nullable).
- Add `marhalah_year` (integer, indexed).
- Add `phone_number` (string, nullable).
- Add `is_verified` (boolean, default: false).

### D. Global Scoping Logic

Create a Global Scope `app/Models/Scopes/MarhalahScope.php`:

- **Logic:** If `config('community.scope') === 'single'`, filter all queries to only show users where `marhalah_year` matches `config('community.target_marhalah.year')`.
- Apply this scope to the `User` model using the `booted` method.
