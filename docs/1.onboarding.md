# Task 2: Authentication & Onboarding Flow

## 1. Context & Business Logic
The application utilizes a **Google Account Only** authentication strategy ("One Button Rule"). This eliminates password liability and OTP management. 

### User Journey:
1. User clicks "Login with Google" on the Landing Page.
2. System checks the database via Google's callback email:
   * **If Email Exists:** Update avatar/name (if changed) and redirect to the Dashboard.
   * **If Email is New:** Redirect to the **Onboarding Form**.
3. **Onboarding Form:** Name and Email are pre-filled (read-only). The user must input:
   * **Marhalah (Graduation Year):** If `config('community.scope') === 'single'`, this field is hidden/read-only and defaults to the configured year. If `global`, show a dropdown.
   * **Kampus / Dormitory:** (e.g., Gontor 1, Gontor 2).
   * **WhatsApp Number:** Mandatory for communication.
4. **Waiting Room Concept:** Upon form submission, the user is created with `is_verified = false`. They can access the portal but restricted from viewing sensitive data (like other users' phone numbers) until verified by an Admin or Peer.

---

## 2. Implementation Task for AI Agent
**Instruction for AI Agent:** Generate the Auth Controllers, Routes, and React/Inertia frontend components to handle the Google Socialite flow and the Onboarding process.

### A. Backend: Routes & Socialite Controller
1. **Routes:** Create endpoints for Google redirect (`/auth/google/redirect`) and callback (`/auth/google/callback`).
2. **Auth Controller (`App\Http\Controllers\Auth\GoogleAuthController`):**
   * Use `Socialite::driver('google')->user()`.
   * Check if the user exists. 
   * If exists: `Auth::login($user)` -> redirect to `/dashboard`.
   * If new: Store the Google User data temporarily in the Session and redirect to `/onboarding`.

### B. Backend: Onboarding Controller
1. **Controller (`App\Http\Controllers\Auth\OnboardingController`):**
   * `show()`: Render the Inertia `Auth/Onboarding` React component. Pass the Google user data from the session and the `community.scope` config as props.
   * `store()`: Validate the incoming request (WhatsApp, Kampus, Marhalah). Create the user in the database with `google_id`, `is_verified = false`, and log them in. Clear the session. Redirect to `/dashboard`.

### C. Frontend: React + Inertia Components
1. **Landing Page / Login UI (`Pages/Auth/Login.jsx`):**
   * A clean, centered UI.
   * Primary Action: A large button "Gabung dengan Google" (Join with Google) utilizing the "Ijo Kukus" (muted green) color palette.
2. **Onboarding Form UI (`Pages/Auth/Onboarding.jsx`):**
   * **Pre-filled Data:** Name and Email fields (disabled).
   * **Dynamic Marhalah Field:** Read the `communityScope` prop. If `single`, hide or disable the input and set the value to the configured year. If `global`, render a select dropdown.
   * **Input Fields:** WhatsApp Number and Kampus.
   * Apply Shadcn UI form components with Tailwind CSS (muted green accents).

---

## 3. Definition of Done (DoD)
- [ ] User can click the Google login button and successfully authenticate via Google OAuth.
- [ ] Existing users are redirected immediately to the Dashboard.
- [ ] New users are intercepted and redirected to the Onboarding React component.
- [ ] The Onboarding form handles the `COMMUNITY_SCOPE` correctly for the Marhalah field.
- [ ] New user is successfully saved to the database with `is_verified = false` and logged in.
